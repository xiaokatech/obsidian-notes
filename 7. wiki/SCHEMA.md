# Wiki Schema — 配置文档

> 这份文档告诉 LLM 这个知识库的结构规则和工作流。  
> 随着知识库的演进，你和 LLM 共同更新这份文档。

---

## 笔记库整体结构

```
obsidian-notes/
  1. 索引/          ← 全局入口，总索引
  1.2 主题/         ← 主题索引页（内容创作、工作、项目）
  1.2a 项目/        ← 项目索引页
  2. 模板/          ← Obsidian 模板

  # --- 原始来源层（LLM 只读）---
  3.1 笔记-note/    ← 技术笔记、读书笔记、学习卡片（重要原始来源）
  4.1 附件/         ← 媒体文件（图片、PDF 等）
  5.1 日记/          ← 日记、思考记录、个人成长轨迹（重要原始来源）
  5.2 创作/          ← 基于原始来源，wiki 内容产生的创作
  6. 原始来源/      ← 外部采集的来源集合
    文章/           ← 外部文章、博客、报道
    会议记录/       ← 工作讨论、会议纪要
    其他/           ← 其他类型原始材料

  # --- LLM 知识合成层（LLM 完全拥有）---
  7. wiki/          ← LLM 维护的结构化知识库
    index.md        ← wiki 总目录（按内容组织）
    log.md          ← 操作日志（按时间组织）
    SCHEMA.md       ← wiki 规则与工作流配置
    实体/           ← 人物、公司、产品、项目页面
    概念/           ← 方法论、框架、理论、行业逻辑页面
    综合分析/       ← 跨来源综合、对比分析、研究报告

```

## LLM 工作规则

### 原始来源层（LLM 只读）

这些是 LLM 消化知识的原材料，不可修改，只可读取分析：

**A. 知识卡片与学习笔记** 📚

- `3.1 笔记-note/` — 技术笔记、商业分析、读书笔记、学习卡片
    - 特点：结构化笔记，包含 Context/What/Why/How 等逻辑框架
    - 例子：Terraform 技术对比、商业模式分析、论文阅读笔记
    - LLM 处理：提取核心概念，创建实体和概念页面

**B. 个人思考与成长轨迹** 📝

- `5. 日记/` — 日记、工作复盘、个人成长记录
    - 特点：按时间组织的个人思考，包含计划、反思、感悟
    - 例子：每日计划、项目想法、自我认知变化
    - LLM 处理：发现模式趋势，建立个人成长综合分析

**C. 外部采集来源** 🌐

- `6. 原始来源/` — 外部文章、会议记录、研究报告等
    - 特点：从外部世界收集的原始信息
    - 例子：技术博客、商业报道、会议纪要
    - LLM 处理：摘要核心观点，融入现有知识结构

### 知识合成层（LLM 可写）

这些是 LLM 生成和维护的结构化知识库：

- `7. wiki/` 下的所有文件 ✅ — LLM 完全拥有
- `SCHEMA.md` ✅ — 与用户协商后更新
- `1. 索引/总索引.md` ✅ — 更新 wiki 入口链接

## 消化来源（Ingest）工作流

当用户说"消化这篇文章/笔记/来源"时，执行以下步骤：

1. **阅读来源** — 读取文件内容
2. **与用户确认要点** — 简要说明提取到的关键信息，问是否有补充
3. **写摘要页面** — 在 `7. wiki/` 根目录或对应子目录创建摘要文件
4. **更新实体/概念页面** — 检查来源涉及的实体和概念，更新或新建对应页面
5. **更新 index.md** — 在总索引中加入新页面的条目
6. **追加 log.md** — 记录本次消化

### 摘要页面命名规则

`YYYY-MM-DD-摘要-[来源标题简写].md`

### 实体页面模板

```markdown
# [实体名称]

#实体 #wiki

## 基本信息

- 类型：（人物 / 公司 / 产品 / 项目）
- 关联主题：

## 核心描述

（2-3 句话说清楚这个实体是什么）

## 关键信息

（结构化的重要内容）

## 来源

- [[来源文件名]] — 摘要说明
```

## 查询（Query）工作流

当用户提问时：

1. 先读 `7. wiki/index.md` 找相关页面
2. 读取相关 wiki 页面
3. 综合回答，标注引用来源
4. 如果回答本身有价值，询问用户是否存为综合分析页面

---

## 检查（Lint）工作流

定期执行，检查：

- [ ] 有无页面间矛盾
- [ ] 有无提及但没有独立页面的重要概念/实体
- [ ] 有无孤立页面（无入链）
- [ ] 有无过时结论需要更新
- [ ] index.md 是否完整
